# RAG 시스템용 기업 메신저 작성 가이드라인

## 1. 메신저 커뮤니케이션의 5대 핵심 원칙 (5 Core Principles)

이 가이드라인은 사내 메신저(슬랙, 팀즈, 카카오워크 등) 및 업무용 채팅 도구에서의 효율적이고 전문적인 커뮤니케이션을 위한 기준입니다. LLM은 이 원칙을 기반으로 메시지를 평가하고 개선안을 제안해야 합니다.

1. **즉시성 (Immediacy):** 메신저는 실시간 커뮤니케이션 도구이므로, 긴급도와 중요도를 고려하여 적절한 시점에 신속하게 전달해야 합니다.

2. **간결성 (Conciseness):** 핵심 내용만 담아 짧고 명확하게 작성합니다. 긴 내용은 문단을 나누거나 별도 문서로 공유합니다.

3. **맥락성 (Context-Awareness):** 채널의 성격과 참여자를 고려하여 적절한 톤앤매너와 정보 수준을 선택합니다.

4. **존중성 (Respectfulness):** 동료 간 상호 존중을 바탕으로 예의를 지키되, 과도한 격식은 피하여 자연스럽게 소통합니다.

5. **추적성 (Traceability):** 중요한 업무 내용은 기록이 남고 검색 가능하도록 명확한 키워드와 구조로 작성합니다.

## 2. 채널별 메시지 작성 기준 (Channel-Specific Guidelines)

### 2.1 공개 채널 (Public Channels)
**목적:** 팀 전체 또는 부서 전체가 알아야 할 정보 공유

**작성 원칙:**
- 모든 구성원이 이해할 수 있는 명확한 표현 사용
- 개인적인 내용보다는 업무 관련 내용 위주
- 중요한 공지사항은 @channel 또는 @here 활용

**좋은 예시:**
[공지] 내일(9/29) 오전 10시 전체 회의 있습니다

장소: 2층 대회의실
주제: Q3 성과 리뷰 및 Q4 계획 수립
준비사항: 각 팀별 분기 보고서


**나쁜 예시:**
내일 회의 있어요~ 다들 오세요!

### 2.2 개인 메시지 (Direct Messages)
**목적:** 1:1 업무 협의, 개인적인 문의, 민감한 내용 논의

**작성 원칙:**
- 상대방의 업무 시간과 상황 고려
- 긴급하지 않은 경우 인사말로 시작
- 구체적인 요청사항 명시

**좋은 예시:**
안녕하세요!
마케팅 자료 검토 관련해서 질문이 있어서 연락드려요.
15페이지 예산 부분에서 A항목 산출근거를 추가로 받을 수 있을까요?
급하지 않으니 편한 시간에 답변 부탁드려요 😊

### 2.3 프로젝트 채널 (Project Channels)
**목적:** 특정 프로젝트 관련 업무 진행사항 공유 및 협업

**작성 원칙:**
- 프로젝트 관련성이 높은 내용만 게시
- 진행상황, 이슈, 의사결정 사항 명확히 구분
- 액션 아이템과 담당자, 마감일 명시

**좋은 예시:**
[진행상황] 웹사이트 리뉴얼 프로젝트 업데이트
✅ 완료: 와이어프레임 1차 검토 (9/25)
🔄 진행 중: UI 디자인 작업 (김디자님, 예정일: 10/2)
⚠️ 이슈: 서버 용량 검토 필요 → @박개발님 확인 요청
📋 다음 단계: 사용성 테스트 일정 조율

## 3. 메시지 구조 및 작성법 (Message Structure & Writing Tips)

### 3.1 기본 메시지 구조
1. **인사/호명** (필요시)
2. **핵심 내용** (목적과 요청사항)
3. **상세 정보** (배경, 첨부자료 등)
4. **액션 아이템** (누가, 언제, 무엇을)
5. **마무리** (감사인사 등)

### 3.2 효과적인 메시지 작성 기법

**가독성 향상:**
- 긴 메시지는 줄바꿈으로 단락 구분
- 중요한 정보는 **굵게** 또는 `코드블록` 활용
- 목록은 • 또는 - 기호 사용

**이모지 활용:**
- 감정 표현과 친근감 조성: 😊 👍 🎉
- 상태 표시: ✅ (완료), 🔄 (진행중), ⚠️ (주의)
- 과도한 사용은 지양, 업무 맥락에 적합한 선택

**멘션 활용:**
- @개인명: 특정 사람에게 직접 요청/확인
- @here: 현재 온라인인 채널 구성원
- @channel: 모든 채널 구성원 (긴급/중요시에만)

### 3.3 시간대별 메시지 에티켓

**업무시간 (09:00~18:00):**
- 일반적인 업무 관련 메시지는 자유롭게 전송
- 긴급하지 않은 경우에도 적절한 시점에 전달

**업무시간 외 (야간/주말):**
- 긴급한 경우에만 전송
- "급하지 않으니 내일 확인하셔도 됩니다" 등 배려 문구 추가
- 예약 전송 기능 활용 권장

## 4. 상황별 권장 및 지양 표현 (Dos & Don'ts)

| 상황 | 권장 표현 | 지양 표현 |
|------|-----------|-----------|
| 요청사항 | 확인 부탁드려요 / 검토 요청드려요 | 이거 좀 봐줘 / 빨리 해줘 |
| 감사표현 | 감사해요! / 고생하셨어요! | ㄳㄳ / 수고 |
| 확인응답 | 네, 확인했습니다 / 알겠습니다! | ㅇㅇ / ㅋㅋ / 넵 |
| 의견제시 | ~는 어떠세요? / 제안드리면 ~ | 그건 아닌듯 / 당연히 ~ |
| 거절/연기 | 죄송하지만 일정상 어려울 것 같아요 | 안 돼요 / 못해요 |
| 첨부파일 | 파일 첨부했어요! / 링크 공유드려요 | 파일 / 이거 |

## 5. 표준 메시지 템플릿 (Standard Message Templates)

아래는 본 가이드라인을 기반으로 작성된 표준 메시지 양식입니다. 기업별 특화된 양식이 없을 경우, 이 양식을 기본으로 사용하거나 참고하여 작성하는 것을 권장합니다.

### 5.1 업무 요청 메시지
안녕하세요! [이름]님
[구체적인 요청사항]에 대해 도움이 필요해서 연락드려요.
📋 요청내용:

[세부사항 1]
[세부사항 2]

⏰ 희망 완료일: [날짜]
📎 관련자료: [첨부파일/링크]
급하지 않으니 가능한 시간에 확인 부탁드려요!
감사합니다 😊

### 5.2 진행상황 공유 메시지
[프로젝트명] 주간 진행상황 공유
✅ 이번 주 완료:

[완료사항 1]
[완료사항 2]

🔄 다음 주 계획:

[계획사항 1] - @담당자명
[계획사항 2] - @담당자명

⚠️ 이슈/리스크:

[이슈사항] → 해결방안: [방안]

질문이나 의견 있으시면 언제든 말씀해주세요!

### 5.3 회의 관련 메시지
📅 [회의명] 일정 안내

일시: [날짜] [시간]

참석 확인 부탁드려요! 👍

### 5.4 긴급 상황 메시지

문의사항은 @담당자명으로 연락 부탁드려요.

## 6. 메신저 사용 시 주의사항 (Important Considerations)

### 6.1 보안 관련
- 민감한 정보 (개인정보, 기밀사항)는 보안 채널 또는 별도 문서 활용
- 외부 링크 첨부 시 보안성 검토
- 스크린샷 공유 시 개인정보 모자이크 처리

### 6.2 업무 효율성
- 같은 내용을 여러 채널에 중복 게시 지양
- 스레드(Thread) 기능 활용으로 대화 맥락 유지
- 중요한 결정사항은 별도 문서로 기록 보관

### 6.3 조직 문화 고려
- 회사별 메신저 사용 정책 우선 준수
- 팀 내 소통 규칙과 문화 존중
- 세대 간, 직급 간 소통 방식 차이 고려

## 7. 체크리스트 (Message Quality Checklist)

메시지 전송 전 다음 사항을 확인하세요:

**내용 검증:**
- [ ] 전달하고자 하는 핵심 내용이 명확한가?
- [ ] 상대방이 이해하기 쉽게 작성되었는가?
- [ ] 필요한 정보(일시, 장소, 담당자 등)가 포함되었는가?

**형식 검증:**
- [ ] 맞춤법과 띄어쓰기가 정확한가?
- [ ] 적절한 존댓말과 호칭을 사용했는가?
- [ ] 이모지나 특수문자가 과하지 않은가?

**수신자 검증:**
- [ ] 적절한 채널에 게시하는가?
- [ ] 멘션이 필요한 사람들이 포함되었는가?
- [ ] 전송 시간이 적절한가?

**액션 검증:**
- [ ] 상대방이 해야 할 일이 명확한가?
- [ ] 마감일이나 우선순위가 표시되었는가?
- [ ] 후속 조치 계획이 있는가?
